# Dimension 2: Usability Risk

Can users figure it out and get value?

## Criteria

- **U1: User Testing** — Has a prototype or mockup been tested with real users?
- **U2: Learning Curve** — Is the expected learning curve assessed and acceptable?
- **U3: Workflow Fit** — Does it fit existing user workflows without disruption?
- **U4: Accessibility** — Are accessibility requirements addressed for target platforms?

## Proposal Questions

1. Has a prototype been tested with real users? How many? (U1)
2. What is the expected learning curve? Can users succeed without training? (U2)
3. How does this fit into users' existing workflows and tools? (U3)
4. What accessibility requirements apply, and are they addressed? (U4)

## Implementation — Compliance Questions

1. Was usability tested before building? (U1)
2. Was a design review conducted with UX expertise? (U1, U3)
3. Were target personas defined with specific needs? (U2)

## Implementation — Results Questions

1. What are task completion rates for the core workflow? (U1, U2)
2. Where are users dropping off in the funnel? (U3)
3. What patterns appear in support tickets? (U2)
4. Can new users succeed without training or documentation? (U2)

## Scoring

| Score | Description |
|-------|-------------|
| 5 | Prototype tested with 5+ representative users; friction points identified and resolved; accessibility verified |
| 4 | Usability testing done with some gaps (e.g., fewer users, or accessibility not yet verified) |
| 3 | Design review but no user testing; assumed to be intuitive |
| 2 | Design by engineers without UX input; "we'll iterate after launch" |
| 1 | No usability consideration; user experience not discussed |

## Quality Check

A score of 4+ requires at minimum: prototype or mockup tested with real users, documented friction points and resolutions, and accessibility requirements identified.
