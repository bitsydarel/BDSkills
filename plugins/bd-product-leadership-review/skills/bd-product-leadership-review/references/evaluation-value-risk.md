# Dimension 1: Value Risk

Are we solving a real problem customers care about?

## Criteria

- **V1: Problem Specificity** — Is a specific customer problem identified with concrete language?
- **V2: Demand Evidence** — Is there evidence of demand beyond opinion or stakeholder assertion?
- **V3: Need Intensity** — Is this a must-have or nice-to-have? Would customers miss it if removed?
- **V4: Segment Definition** — Is the target segment defined and sized?
- **V5: Alternative Assessment** — Are current alternatives and switching costs understood?

## Proposal Questions

1. What specific problem does this solve? (V1)
2. What evidence exists that customers want or need this? (V2)
3. Is this a must-have or nice-to-have for the target segment? (V3)
4. Who is the target customer, and how large is the segment? (V4)
5. What are customers using today, and why would they switch? (V5)

## Implementation — Compliance Questions

1. Was a clear problem defined before building? (V1)
2. Was demand validated with actual customers? (V2)
3. Was the target segment documented and scoped? (V4)

## Implementation — Results Questions

1. Did customers adopt it? What does usage data show? (V2, V3)
2. What are retention and engagement metrics? (V3)
3. Did it solve the stated problem based on customer feedback? (V1)
4. Would customers miss it if removed? (V3)

## Scoring

| Score | Description |
|-------|-------------|
| 5 | Multiple interviews confirm problem + quantitative demand signal + clear segment definition + alternatives mapped |
| 4 | Customer evidence exists with minor gaps (e.g., segment sized but alternatives not fully mapped) |
| 3 | Anecdotal evidence; problem assumed from observation; segment loosely defined |
| 2 | No direct customer contact; relying on stakeholder assertions or competitor moves |
| 1 | No customer contact; solution looking for a problem |

## Quality Check

A score of 4+ requires at minimum: a named customer segment, a quantified demand signal (interviews, survey data, usage metrics), and a documented problem statement using customer language.
